# JavaScript Learning Journey

This repository documents my raw learning journey, experiments, and deep dives into JavaScript. It contains a structured collection of practice scripts, breakdown notes, and modular exercises compiled while following comprehensive courses from both **Chai aur Code** and **Tap Academy**, alongside self-directed documentation research.

The primary objective of this repository is to move systematically from basic imperative logic to advanced functional programming and engine architecture.

---

## Repository Structure

The curriculum is organized into explicit execution modules for efficient navigation and concept lookup.

### Foundations and Engine Mechanics
* **01_JS_Basics** - Variables, core data types, and implicit/explicit type conversions.
* **03_Objects** - Mapping real-world data models into structured JavaScript objects.
* **04_Functions** - Building foundational blocks of reusable execution logic.
* **05_JS_BTS** - Behind The Scenes: Execution Contexts, Call Stacks, and V8 engine compilation mechanics.
* **19_Hoisting** - Demystifying the temporal dead zone and how variable/function declarations are allocated in memory.
* **20_Scopes** - Navigating the strict boundary rules of Global, Local, and Block scopes.

### Control Flow and Declarative Logic
* **06_Control_Flow** - Conditional structures, switch cases, and complex logic branching.
* **07_Iterators** - Mastering manual iteration patterns using loops like for, while, and do-while.
* **08_High_Order_Array_Loops** - Transitioning away from manual iterators into built-in pipeline handlers like .forEach, .map, and .filter.
* **09_Filter_Map_Reduce** - Deep array pipeline processing. Condensing data collections into single outcomes without mutating the source array.

### Browser Architecture (DOM Manipulation)
* **10_DOM & 11_DOM_Overview** - Accessing document tree nodes, manipulating styles dynamically, and configuring browser event listeners.
* **11_DOM_Projects** - Practical, real-time interface manipulation and behavioral UI builds.

### Async Engine and API Integration
* **13_Async_JS** - Concurrency mechanics, non-blocking execution flow, timers, and interval loops.
* **14_API_Request** - Working with legacy XMLHttpRequests to understand data transfer primitives.
* **15_JSON_API** - Consuming modern REST payloads and using raw JSON data formats.
* **16_Asynchronous_Programming** - Balancing synchronous vs. asynchronous system executions.
* **17_Promises** - Managing microtask processing, promise state machines, and handling error exceptions through .catch blocks.

### Advanced Programming Paradigms
* **18_Functions_Deep** - Advanced lexical abstractions, closures, and self-invoking scopes.
* **21_OOP** - Object-Oriented Programming, constructor blueprints, prototype inheritance, and modern ES6 classes.
* **22_Functions_As_Objects** - Evaluating functions as first-class citizens and tracking how they inherit prototype properties.
* **23_Functional_Programming** - Practical implementation of functional paradigms: data immutability, function purity, and composition mechanics.

### Practical Labs
* **Projects Folder** - Standalone application mini-builds and random architectural sandboxes.

3. **Reference Hub** - Maintaining a zero-boilerplate codebase to reference and safely adapt during real production work.
